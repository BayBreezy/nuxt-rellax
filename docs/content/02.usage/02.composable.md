`useRellax` is auto-imported by the module. It accepts a target element (selector string, `Element`, or a Vue template ref) and returns a stable API object you can use to control the instance from script.

## Signature

```ts
function useRellax(
  el?: string | Element | Ref<HTMLElement | null | undefined>,
  options?: UseRellaxOptions | (() => UseRellaxOptions),
): {
  instance: Ref<RellaxInstance | null>
  refresh: () => void
  destroy: () => void
  reinitialize: () => void
}
```

---

## Targeting Elements

### Template ref (recommended)

Pass a `useTemplateRef` directly. The composable watches the ref, so if the element is toggled with `v-if`, Rellax is automatically destroyed and re-created.

```vue [ParallaxCard.vue]
<script setup lang="ts">
const el = useTemplateRef<HTMLElement>('card')

const { refresh, reinitialize } = useRellax(el, { speed: -3 })
</script>

<template>
  <div ref="card" class="parallax-card">
    <slot />
  </div>
</template>
```

### CSS selector

Targets all elements matching the selector — useful when Rellax manages a class of elements.

```vue [ParallaxGrid.vue]
<script setup lang="ts">
useRellax('.parallax-item', { speed: -2, center: true })
</script>

<template>
  <div class="grid">
    <div class="parallax-item">A</div>
    <div class="parallax-item">B</div>
    <div class="parallax-item">C</div>
  </div>
</template>
```

### Direct Element reference

```ts
const el = document.querySelector<HTMLElement>('#hero-bg')!
useRellax(el, { speed: -4 })
```

## Using the Return Value

```vue [ControlledParallax.vue]
<script setup lang="ts">
const el = useTemplateRef<HTMLElement>('bg')

const { instance, refresh, destroy, reinitialize } = useRellax(el, { speed: -2 })

// Read the raw Rellax instance (null until mounted)
watchEffect(() => {
  if (instance.value) {
    console.log('Rellax is active')
  }
})

// Recompute positions after a layout change
function onLayoutChange() {
  refresh()
}

// Fully tear down parallax (e.g. in response to user preference)
function disableParallax() {
  destroy()
}

// Destroy and re-create with the latest options
function resetParallax() {
  reinitialize()
}
</script>
```

## Reactive Options via Getter

Pass a getter function `() => options` instead of a plain object. `reinitialize()` will then always use the **latest** value, not a snapshot from mount time.

```vue [ConfigurableParallax.vue]
<script setup lang="ts">
const speed = ref(-2)
const center = ref(false)

const el = useTemplateRef<HTMLElement>('el')

const { reinitialize } = useRellax(el, () => ({
  speed: speed.value,
  center: center.value,
}))

// Changing speed and calling reinitialize() picks up the new value
async function applyNewSpeed(newSpeed: number) {
  speed.value = newSpeed
  await nextTick()
  reinitialize()
}
</script>

<template>
  <div ref="el">Parallax element</div>
  <input v-model.number="speed" type="range" min="-10" max="10" @change="reinitialize" />
</template>
```

## Intersection Observer Mode

Set `observeIntersection: true` to lazy-initialize the Rellax instance — it activates when the element enters the viewport and pauses when it leaves. This is a good performance win on long pages with many parallax elements.

```vue [LazyParallax.vue]
<script setup lang="ts">
const el = useTemplateRef<HTMLElement>('hero')

useRellax(el, {
  speed: -3,
  observeIntersection: true,

  // Fine-tune the IntersectionObserver
  intersectionThreshold: 0.1,
  intersectionRootMargin: '100px',
})
</script>

<template>
  <div ref="hero" class="hero-section">
    <slot />
  </div>
</template>
```

::prose-callout{variant="info" title="Observation target"}
`observeIntersection` requires the target to be an `Element` or a `Ref<HTMLElement>`. It does **not** work with a CSS selector string (which can match multiple elements).
::

## Custom Scroll Wrapper

```vue [ScrollContainerParallax.vue]
<script setup lang="ts">
const el = useTemplateRef<HTMLElement>('item')

useRellax(el, {
  speed: -2,
  wrapper: '#my-scroll-container',
  relativeToWrapper: true,
})
</script>

<template>
  <div id="my-scroll-container" class="h-[400px] overflow-y-auto">
    <div ref="item" class="tall-content">
      Scrolls relative to the container above
    </div>
  </div>
</template>
```

## v-if Toggling

When you pass a template ref, `useRellax` watches it with Vue's `watch`. If `v-if` removes the element, Rellax is destroyed. When `v-if` brings it back, Rellax is re-created.

```vue [ToggleParallax.vue]
<script setup lang="ts">
const show = ref(true)
const el = useTemplateRef<HTMLElement>('el')

// Rellax auto-destroys when show = false, auto-creates when show = true
useRellax(el, { speed: -3 })
</script>

<template>
  <button @click="show = !show">Toggle</button>
  <div v-if="show" ref="el" class="parallax-box">Content</div>
</template>
```

## SSR

On the server, `useRellax` returns a stub with no-op functions and a `null` instance. No `typeof window` guards needed in your component.

```ts
// Safe to call at the top level of setup() — no SSR errors
const { instance } = useRellax(el, { speed: -2 })
```

## Return Value

:::prose-field-group{variant="bordered"}
::prose-field{name="instance" type="Ref<RellaxInstance | null>"}
The raw Rellax instance. `null` on the server and before the component has mounted.
::
::prose-field{name="refresh" type="() => void"}
Calls `instance.refresh()`. Recomputes positions without destroying and re-creating the instance. Use this after DOM layout changes.
::
::prose-field{name="destroy" type="() => void"}
Destroys the Rellax instance and sets `instance` to `null`. The element returns to its natural scroll position.
::
::prose-field{name="reinitialize" type="() => void"}
Calls `destroy()` then `initialize()` in sequence. If options were provided as a getter, the latest return value is used for the new instance.
::
:::
