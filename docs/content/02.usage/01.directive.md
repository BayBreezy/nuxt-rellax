`v-rellax` is automatically registered by the module. Attach it to any element and optionally pass a Rellax options object. Module-level defaults are merged in automatically.

## Basic Usage

```vue
<template>
  <!-- Uses module defaults (or Rellax defaults if none configured) -->
  <div v-rellax>Parallax element</div>

  <!-- Override speed -->
  <div v-rellax="{ speed: -3 }">Slower scroll up</div>

  <!-- Positive speed scrolls downward -->
  <div v-rellax="{ speed: 2 }">Drifts down</div>
</template>
```

::prose-callout{variant="info" title="Speed reference"}
Speed ranges from `-10` to `10`. Negative values move the element **upward** relative to the page scroll. Positive values move it **downward**. Speed `0` disables parallax on that element.
::

## Modifiers

Modifiers are a shorthand for common Rellax boolean options. They can be combined with a value object.

### `.center`

Centers the parallax effect — the element starts at its natural position and the parallax offset is applied symmetrically as the user scrolls toward and away from it.

```vue
<template>
  <div v-rellax.center>Centered parallax</div>

  <!-- Combined with speed -->
  <div v-rellax.center="{ speed: -4 }">Centered + fast</div>
</template>
```

### `.horizontal`

Applies the parallax offset on the **horizontal axis** instead of the vertical axis.

```vue
<template>
  <div v-rellax.horizontal="{ speed: 3 }">Drifts sideways</div>
</template>
```

### Combining modifiers

```vue
<template>
  <!-- Centered horizontal parallax -->
  <div v-rellax.center.horizontal="{ speed: -2 }">
    Centered horizontal drift
  </div>
</template>
```

## Common Patterns

### Hero background

A classic parallax hero where the background moves at half the scroll speed, creating depth.

```vue [pages/index.vue]
<template>
  <section class="relative h-screen overflow-hidden">
    <!-- Background moves slower than the page -->
    <div
      v-rellax="{ speed: -4 }"
      class="absolute inset-0 -top-[20%] -bottom-[20%]"
    >
      <img src="/hero-bg.jpg" class="h-full w-full object-cover" alt="" />
    </div>

    <!-- Foreground content scrolls normally -->
    <div class="relative z-10 flex h-full items-center justify-center">
      <h1 class="text-white text-6xl font-bold">Hello World</h1>
    </div>
  </section>
</template>
```

### Custom scroll wrapper

By default Rellax tracks the window scroll. You can scope it to a specific scrollable container using `wrapper` and `relativeToWrapper`.

```vue [ScrollableSection.vue]
<template>
  <div id="scroll-container" class="h-[500px] overflow-y-auto">
    <div
      v-rellax="{
        wrapper: '#scroll-container',
        relativeToWrapper: true,
        speed: -3,
      }"
    >
      Parallax inside a custom scroller
    </div>
  </div>
</template>
```

### Multiple layers

Stack several elements at different speeds to build a multi-layer parallax scene.

```vue [ParallaxScene.vue]
<template>
  <div class="scene">
    <div v-rellax="{ speed: -6 }" class="layer layer-back">Sky</div>
    <div v-rellax="{ speed: -4 }" class="layer layer-mid">Mountains</div>
    <div v-rellax="{ speed: -2 }" class="layer layer-front">Trees</div>
    <div class="layer layer-ground">Ground</div>
  </div>
</template>
```

### Centering on a specific element

```vue [CardSection.vue]
<template>
  <!--
    center: true makes the element's rest position its natural position
    and offsets symmetrically — great for cards in a scrolling feed.
  -->
  <div
    v-for="card in cards"
    :key="card.id"
    v-rellax.center="{ speed: -2 }"
    class="card"
  >
    {{ card.title }}
  </div>
</template>
```

## Updating Options

When the component re-renders, the directive calls `rellaxInstance.refresh()` automatically. You do **not** need to destroy and recreate the instance when data changes.

```vue [DynamicSpeed.vue]
<script setup lang="ts">
const speed = ref(-2)
// When speed changes and the component re-renders,
// the directive calls refresh() internally.
</script>

<template>
  <div v-rellax="{ speed }">Dynamic speed</div>
</template>
```

::prose-callout{variant="warning" title="Changing the options object itself"}
The directive calls `refresh()` on every component update, which recomputes positions. If you need to change options that `refresh()` cannot update (like `wrapper` or `horizontal`), you will need to use `useRellax` instead, which exposes a `reinitialize()` method.
::

---

## SSR

`v-rellax` is fully SSR-safe. On the server it returns an empty `getSSRProps()` object, so no hydration errors occur. The Rellax instance is only created after the component mounts in the browser.

---

## Directive API

| Hook | Behaviour |
|---|---|
| `mounted` | Creates a new `Rellax` instance on the element, merging module defaults with the binding value |
| `updated` | Calls `instance.refresh()` to recompute positions |
| `unmounted` | Calls `instance.destroy()` and removes the reference |
