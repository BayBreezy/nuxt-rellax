`useRellaxGroup` is auto-imported by the module. It creates multiple `useRellax` instances from an array of targets and returns a single object to `refresh`, `destroy`, or `reinitialize` all of them at once.


## Signature

```ts
function useRellaxGroup(
  items: Array<
    | string
    | Element
    | Ref<HTMLElement | null | undefined>
    | { el: string | Element | Ref<HTMLElement | null | undefined>; options?: UseRellaxOptions }
  >,
  sharedOptions?: UseRellaxOptions,
): {
  instances: RellaxReturn[]
  refresh: () => void
  destroy: () => void
  reinitialize: () => void
}
```

---

## Basic Usage

### Shared options across all elements

Pass an array of CSS selectors with a single options object applied to every target.

```vue [ParallaxSection.vue]
<script setup lang="ts">
// All three elements get speed: -2
useRellaxGroup(['.bg-layer', '.mid-layer', '.fg-layer'], { speed: -2 })
</script>

<template>
  <div class="scene">
    <img class="bg-layer" src="/bg.jpg" alt="" />
    <img class="mid-layer" src="/mid.png" alt="" />
    <img class="fg-layer" src="/fg.png" alt="" />
  </div>
</template>
```

### Per-element options

Wrap each target in `{ el, options }` to give it its own Rellax options. `sharedOptions` (the second argument) are merged first, then per-element options override them.

```vue [MultiLayerParallax.vue]
<script setup lang="ts">
useRellaxGroup([
  { el: '.sky',       options: { speed: -1 } },
  { el: '.clouds',    options: { speed: -3 } },
  { el: '.mountains', options: { speed: -5, center: true } },
  { el: '.trees',     options: { speed: -7 } },
])
</script>

<template>
  <div class="landscape">
    <div class="sky" />
    <div class="clouds" />
    <div class="mountains" />
    <div class="trees" />
  </div>
</template>
```

### Mixed: template refs and selectors

You can mix target types freely in the same array.

```vue [MixedGroup.vue]
<script setup lang="ts">
const heroEl = useTemplateRef<HTMLElement>('hero')
const bannerEl = useTemplateRef<HTMLElement>('banner')

useRellaxGroup([
  { el: heroEl,   options: { speed: -4 } },
  { el: bannerEl, options: { speed: -2, center: true } },
  '.floating-card',
], { round: true })
// round: true is merged into all three
</script>

<template>
  <div ref="hero" class="hero" />
  <div ref="banner" class="banner" />
  <div class="floating-card" />
</template>
```

## Controlling the Group

The returned object mirrors the single-instance API but applies to every element in the group.

```vue [ParallaxPage.vue]
<script setup lang="ts">
const { instances, refresh, destroy, reinitialize } = useRellaxGroup(
  ['.layer-a', '.layer-b', '.layer-c'],
  { speed: -3 },
)

// Refresh all after a layout change
function onResize() {
  refresh()
}

// Disable all parallax
function disableAll() {
  destroy()
}

// Re-enable all
function enableAll() {
  reinitialize()
}

// Access individual instances
const firstInstance = computed(() => instances[0].instance.value)
</script>
```

## With `v-for`

`useRellaxGroup` works well alongside `v-for` when you have a dynamic number of elements. Use template refs collected from the loop.

```vue [CardGrid.vue]
<script setup lang="ts">
const cards = ref([
  { id: 1, title: 'Card A', speed: -2 },
  { id: 2, title: 'Card B', speed: -4 },
  { id: 3, title: 'Card C', speed: -6 },
])

// Collect a ref for each card element
const cardEls = ref<(HTMLElement | null)[]>([])

onMounted(() => {
  // Create the group after refs are populated
  useRellaxGroup(
    cards.value.map((card, i) => ({
      el: cardEls.value[i]!,
      options: { speed: card.speed, center: true },
    })),
  )
})
</script>

<template>
  <div class="grid">
    <div
      v-for="(card, i) in cards"
      :key="card.id"
      :ref="(el) => (cardEls[i] = el as HTMLElement)"
      class="card"
    >
      {{ card.title }}
    </div>
  </div>
</template>
```

::prose-callout{variant="tip" title="Simpler alternative for v-for"}
When every element in a loop should have the same parallax behaviour, using `v-rellax` directly on the `v-for` element is simpler than `useRellaxGroup`.

```vue
<div v-for="card in cards" :key="card.id" v-rellax.center="{ speed: -3 }">
  {{ card.title }}
</div>
```
::

## Return Value

:::prose-field-group{variant="bordered"}
::prose-field{name="instances" type="RellaxReturn[]"}
The array of individual `useRellax` return values, in the same order as the input `items` array. You can use this to control specific elements individually.
::
::prose-field{name="refresh" type="() => void"}
Calls `refresh()` on every instance in the group.
::
::prose-field{name="destroy" type="() => void"}
Calls `destroy()` on every instance in the group. All elements return to their natural scroll position.
::
::prose-field{name="reinitialize" type="() => void"}
Calls `reinitialize()` on every instance in the group.
::
:::
