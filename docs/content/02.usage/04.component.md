`<RellaxElement>` is auto-imported by the module. It renders a `div` (or any tag you choose) and manages a Rellax instance internally. Every Rellax option is available as a typed prop, and the instance reinitializes automatically when props change.

## Basic Usage

```vue [Hero.vue]
<template>
  <!-- Renders a <div> with Rellax applied -->
  <RellaxElement :speed="-3" class="hero-bg">
    <img src="/bg.jpg" alt="" />
  </RellaxElement>
</template>
```

---

## Custom Tag

Use the `tag` prop to render any HTML element. The component passes all unknown attributes (`$attrs`) through to the rendered element.

```vue [Hero.vue]
<template>
  <!-- Renders as <section> -->
  <RellaxElement tag="section" :speed="-2" class="hero-section">
    <h1>Welcome</h1>
  </RellaxElement>

  <!-- Renders as <img> -->
  <RellaxElement tag="img" :speed="-4" src="/bg.jpg" alt="" />

  <!-- Renders as <aside> -->
  <RellaxElement tag="aside" :speed="1">
    Sidebar
  </RellaxElement>
</template>
```

---

## Reactive Props

When any Rellax option prop changes, the component calls `reinitialize()` automatically. No manual intervention needed.

```vue [SpeedSlider.vue]
<script setup lang="ts">
const speed = ref(-2)
const center = ref(false)
</script>

<template>
  <!-- Reinitializes whenever speed or center changes -->
  <RellaxElement :speed="speed" :center="center">
    Dynamic parallax
  </RellaxElement>

  <input v-model.number="speed" type="range" min="-10" max="10" />
  <label>
    <input v-model="center" type="checkbox" />
    Center
  </label>
</template>
```

## Intersection Observer

Enable lazy initialization via the `observe-intersection` prop. The Rellax instance is created when the element enters the viewport and destroyed when it leaves.

```vue [LazySections.vue]
<template>
  <div v-for="section in sections" :key="section.id" class="section">
    <RellaxElement
      :speed="-2"
      :observe-intersection="true"
      :intersection-threshold="0.15"
    >
      <img :src="section.image" alt="" />
    </RellaxElement>
  </div>
</template>
```

## Exposing the API

`<RellaxElement>` exposes the same API as `useRellax` via `defineExpose`. Grab a template ref to call `refresh`, `destroy`, or `reinitialize` from a parent component.

```vue [ParentPage.vue]
<script setup lang="ts">
const rellax = useTemplateRef('rellax')

function forceRefresh() {
  rellax.value?.refresh()
}

function resetEffect() {
  rellax.value?.reinitialize()
}
</script>

<template>
  <RellaxElement ref="rellax" :speed="-3">
    Controlled from parent
  </RellaxElement>

  <button @click="forceRefresh">Refresh</button>
  <button @click="resetEffect">Reset</button>
</template>
```

## Props Reference

:::prose-field-group{variant="bordered"}
::prose-field{name="tag" type="string" default-value="div"}
The HTML tag to render. Any valid HTML element tag name is accepted. Attribute fallthrough (`$attrs`) is enabled, so class, style, and other HTML attributes bind to the rendered element.
::
::prose-field{name="speed" type="number" default-value="undefined"}
Parallax speed from `-10` to `10`. Negative moves the element upward; positive moves it downward relative to scroll. Inherits from module `defaults.speed` if not set.
::
::prose-field{name="center" type="boolean" default-value="undefined"}
Center the parallax effect so the element starts at its natural position and offsets symmetrically.
::
::prose-field{name="wrapper" type="string" default-value="undefined"}
CSS selector of a custom scrollable wrapper element to track instead of the window.
::
::prose-field{name="relativeToWrapper" type="boolean" default-value="undefined"}
Compute the parallax offset relative to the wrapper element instead of the document.
::
::prose-field{name="round" type="boolean" default-value="undefined"}
Round transform values to the nearest pixel for crisper rendering on non-retina screens.
::
::prose-field{name="vertical" type="boolean" default-value="undefined"}
Apply the parallax effect on the vertical axis (default Rellax behavior).
::
::prose-field{name="horizontal" type="boolean" default-value="undefined"}
Apply the parallax effect on the horizontal axis.
::
::prose-field{name="zindex" type="number" default-value="undefined"}
CSS `z-index` applied to the element during scroll.
::
::prose-field{name="observeIntersection" type="boolean" default-value="undefined"}
Use an `IntersectionObserver` to lazy-init the Rellax instance when the element enters the viewport and destroy it when it leaves.
::
::prose-field{name="intersectionThreshold" type="number" default-value="0"}
Fraction of the element that must be visible before the IntersectionObserver fires. Range `0`â€“`1`.
::
::prose-field{name="intersectionRootMargin" type="string" default-value="0px"}
Margin around the viewport used by the IntersectionObserver (e.g. `"100px"` to trigger 100 px before the element enters).
::
:::

## Exposed API

:::prose-field-group{variant="bordered"}
::prose-field{name="instance" type="Ref<RellaxInstance | null>"}
The underlying Rellax instance. `null` on the server and before mount.
::
::prose-field{name="refresh" type="() => void"}
Recomputes positions without destroying the instance.
::
::prose-field{name="destroy" type="() => void"}
Destroys the instance and resets the element.
::
::prose-field{name="reinitialize" type="() => void"}
Destroys and recreates the instance with the current props.
::
:::
