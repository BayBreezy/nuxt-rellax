All module options are set under the `rellax` key in `nuxt.config.ts`. Every option has a sensible default so you only need to configure what you want to change.

## Example

```ts [nuxt.config.ts] noFormat
export default defineNuxtConfig({
  modules: ['nuxt-rellax'],

  rellax: {
    // Apply these Rellax options to every instance by default
    defaults: {
      speed: -2,
      center: false,
      round: true,
    },

    // Disable parallax on screens narrower than 768 px (mobile)
    disableBelow: 768,

    // Skip init when the user prefers reduced motion (default: true)
    respectReducedMotion: true,
  },
})
```

---

## Options

:::prose-field-group{variant="bordered"}

::prose-field
---
name: defaults
type: Partial<RellaxOptions>
default-value: {}
class: px-4
---
Default Rellax options merged into **every** instance — directive, composable, and component. Per-instance options always take precedence over module defaults.

```ts [nuxt.config.ts]
rellax: {
  defaults: {
    speed: -2,
    center: true,
    round: true,
  }
}
```
::

::prose-field{name="disableBelow" type="number" default-value="0"}
Viewport width in pixels below which parallax is fully disabled. Instances are destroyed when the window shrinks below this threshold and automatically re-created when it grows past it. Set to `0` (the default) to never disable based on screen size.

```ts [nuxt.config.ts]
rellax: {
  // Disable on anything narrower than a tablet
  disableBelow: 768,
}
```
::

::prose-field{name="respectReducedMotion" type="boolean" default-value="true"}
When `true`, all parallax instances are skipped if the user's operating system has **"Reduce Motion"** enabled. This is the recommended accessible default — parallax can cause discomfort for people with vestibular disorders.

```ts [nuxt.config.ts]
rellax: {
  // Opt out only if you handle reduced motion yourself
  respectReducedMotion: false,
}
```
::
:::

## How defaults merge

Module-level `defaults` are the base layer. Any options you pass to the directive, `useRellax`, `useRellaxGroup`, or `<RellaxElement>` are **shallow-merged on top**, with your per-instance values winning any conflicts.

```ts [nuxt.config.ts] noFormat
// Module defaults
rellax: {
  defaults: { speed: -2, round: true }
}
```

```vue [Hero.vue]
<template>
  <!--
    Effective options: { speed: -5, round: true }
    Per-instance speed: -5 overrides the default -2.
    round: true is inherited from module defaults.
  -->
  <div v-rellax="{ speed: -5 }">Fast layer</div>

  <!--
    Effective options: { speed: -2, round: true }
    Both inherited from module defaults.
  -->
  <div v-rellax>Slow layer</div>
</template>
```

## TypeScript

The `ModuleOptions` type is exported from the package if you need it:

```ts [nuxt.config.ts] noFormat
import type { ModuleOptions } from 'nuxt-rellax'

const rellaxConfig: ModuleOptions = {
  defaults: { speed: -3 },
  disableBelow: 768,
}

export default defineNuxtConfig({
  modules: ['nuxt-rellax'],
  rellax: rellaxConfig,
})
```

::prose-callout{variant="info" title="Runtime Config"}
Module options are exposed to the browser as `useRuntimeConfig().public.rellax`. You can read them at runtime if needed, but you should not mutate them — the composables read this config once at setup time.
::
